Multi-modal inclusion engine

The thing that makes OmniBridge different

Voice I/O

lib/voice.ts · Web Speech API

SpeechRecognition for posting, SpeechSynthesis for "read aloud" on every card. Browser-native, zero backend.

Translate + simplify

lib/translate.ts · Gemini 1.5 Flash

Single JSON prompt returns {en, ar, uk, category, simplified} for a Slovak post. Cached in Dexie.

Offline shell + outbox

vite-plugin-pwa · Workbox

App precached. Feed stale-while-revalidate. Writes queued in Dexie and flushed on online.

▼

Data

Client cache ↔ cloud · identical schema

Dexie (IndexedDB)

posts · outbox · translations

Reactive queries via useLiveQuery. Survives refresh, airplane mode, and flaky 3G.

Supabase · Postgres

one "posts" table · RLS open · realtime

title_sk · title_translations jsonb · category · author name + emoji. No auth, no login friction.

Zustand · localStorage

language · profile · online flag

Name + emoji avatar is the entire "identity" layer.

Why PWA, not native

One codebase. No app store. Installs from any browser — the bar for inclusion is "you have a phone", not "you have a Google account".

Why Gemini Flash

Free tier · fast · JSON-mode. One call returns translations and a simplified Slovak version for Mária. No separate NLU step.

v2 pitch

Swap Gemini for NLLB-200 via transformers.js on-device — covers Romani, works with zero network, zero API bill.
